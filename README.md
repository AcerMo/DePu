# Texas Hold'em Online (P2P 联机免服务器版)

这是一个专为手机端和移动设备设计的、支持多人在线联机同时游玩的德州扑克小游戏。

经过架构重构，本游戏已升级为 **P2P (WebRTC 点对点) 联机免服务器版**。这意味着它是一个 **100% 纯静态网页应用**，不需要部署任何后端 Python 或 Node.js 服务器。你可以将它免费部署在 **GitHub Pages** 或 **Netlify** 上，实现永久免费、免维护的公网多人联机。

---

## 核心特性
1. **零服务器成本，永久免费**：利用 WebRTC 技术，所有游戏逻辑、发牌、比牌和数据通讯直接在玩家的浏览器之间发生。
2. **安全防作弊**：虽然数据直连，但系统会将“房主”的浏览器设为权威节点。在摊牌比牌阶段前，玩家的个人底牌仅存于当事人浏览器中，其他人无法通过前端控制台或网络包抓取作弊。
3. **极佳的手机适配**：采用竖屏优先排版，支持手势滑动投注和快捷选择，非常适合微信浏览器、Safari 等手机端单手把玩。
4. **功能完整**：支持 2~8 人同台竞技、自动边池结算、筹码重买、实时聊天室及牌局日志记录。

---

## 联机原理

- **房主 (Host)**：第一个输入某个房间号（如 `1001`）并点击进入的玩家，其浏览器将自动成为该房间的“虚拟服务器”，负责牌局计算与规则判定。
- **加入者 (Client)**：其他输入相同房间号的玩家，浏览器将自动建立到房主浏览器的直连通道，同步游戏数据。

## 本地测试运行

1. 双击打开 `index.html` 即可在浏览器中运行。
2. 或者在项目根目录下通过任何静态网页服务器运行（如 Python 自带的简易服务器）：
   ```bash
   python -m http.server 8000
   ```
   然后用电脑访问 `http://localhost:8000/` 开始测试。

## 线上部署指南 (GitHub Pages 一键免费部署)

因为是纯静态网页，你可以使用 GitHub Pages 进行免费的公网部署：

1. **推送代码**：将项目文件夹下的所有内容推送至你的 GitHub 仓库（例如 `DePu`）。
2. **开启 Pages 服务**：
   * 打开你的 GitHub 仓库页面，点击顶部的 **"Settings"（设置）**。
   * 在左侧菜单栏找到 **"Pages"** 选项。
   * 在 **"Build and deployment" -> "Source"** 处选择 **`Deploy from a branch`**。
   * 在下方 **"Branch"** 处选择 **`main`** 分支，目录选择 **`/ (root)`**，然后点击 **Save（保存）**。
3. **完成联机**：
   * 等待 1 分钟左右，刷新该页面，顶部会出现一个绿色的提示：“Your site is live at `https://你的用户名.github.io/DePu/`”。
   * 打开该链接，即可随时随地与好友分享网址，进行在线德州扑克联机！
